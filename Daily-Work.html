What is it?

A task manager that reads the room.

Most task managers are just lists. They don't know why a task exists, what happened yesterday, or what you were in the middle of when you closed your laptop. Every morning you're starting from scratch.

Daily Indaba is different. It's a system built directly inside Claude Code ‚Äî Anthropic's AI coding assistant ‚Äî that turns a simple markdown file into a living daily briefing. Claude reads it at the start of every session, understands the full context of every task, and is ready to help you move things forward immediately.

No SaaS subscription. No app to install. No GUI. Just a folder, some markdown files, and an AI that knows your day.

"Indaba" is a Zulu and Xhosa word for a gathering ‚Äî a meeting to discuss the matters of the day. That's exactly what this is.

AI forgets between sessions

Claude's context resets when you close the window. Without a system, you spend the first 10 minutes of every session re-explaining your situation.

Task managers miss the why

Todoist, Notion, Asana ‚Äî they hold your tasks but not your context. The nuance, the client history, the constraints ‚Äî all of that lives in your head.

Reporting is manual work

At the end of the day, producing something shareable ‚Äî a dashboard, a status update ‚Äî means starting over. There's no living document that tracks what's done.

How it works

One file. One session. Full context.

The system has four moving parts. Together they create a loop that preserves everything across sessions, generates shareable output automatically, and keeps Claude oriented from the moment you say hello.

01

The daily markdown file is the source of truth

Every working day has one file: YYYY-MM-DD.md. It holds all tasks ‚Äî numbered sequentially, organised by urgency and client ‚Äî plus a running session log, notes, and completed items. This is the single document Claude reads at the start of every session.

02

CLAUDE.md is the system brain

A CLAUDE.md file inside the project folder gives Claude its instructions: what to read at startup, how to handle shortcuts, how to carry tasks forward, when to generate the dashboard. Claude Code automatically loads this file ‚Äî it's how Claude "knows the system" without you explaining it every time.

03

One-word shortcuts control everything

Type tasks to see all tasks. today for priorities only. 5 done to mark task 5 complete. report to generate the HTML dashboard. update to checkpoint the session state. git to commit and push to GitHub. The entire system is keyboard-driven with no menus, no clicks.

04

The HTML dashboard makes it shareable

Type report and Claude generates a polished HTML dashboard from the current task file ‚Äî client cards, status badges, completed tasks, stat counters. Commit and push to GitHub, and it's live on GitHub Pages within seconds. Share the URL with a team member or a manager and they see the exact current state.

Learnings & the magic

What we figured out building this.

This system didn't arrive fully formed. It was built session by session, shortcut by shortcut. These are the discoveries that made the biggest difference.

CLAUDE.md is a superpower most people don't use

Claude Code automatically reads any CLAUDE.md file it finds in a project folder. This means you can encode your entire workflow ‚Äî session startup steps, shortcut definitions, file conventions, even critical rules like "never mark a task complete from a number alone" ‚Äî and Claude follows them without being reminded. It's the difference between an assistant that needs briefing every day and one that already knows the system.

Context window compression is manageable if you trust the files

Long sessions eventually hit Claude's context limit and the conversation gets compressed into a summary. Early on this felt like losing work. But when every task, decision, and log entry is written to files in real time, the compression is harmless. The files survive. Claude reads them again. The session continues without missing a beat.

The distinction between viewing a task and completing it is critical

The most consequential design decision in the shortcut system: a number alone (5) shows task detail; a number plus "done" (5 done) marks it complete. Before this rule was formalised, there were accidental completions. After it was written into CLAUDE.md as a bolded critical rule, it never happened again. Small convention, big reliability gain.

The update checkpoint is more useful than end-of-day saving

We originally thought "save everything at the end of the day." But mid-session checkpoints turned out to be more valuable. The update shortcut ‚Äî verify tasks, write carry-forward notes, update system files, log the state ‚Äî means you can close the laptop at any moment and lose nothing. End of day becomes just a git push.

The session log is more valuable than it looks

It starts as a simple running record. Over time it becomes something more useful: a searchable history of decisions. "When did we decide to move the Sani campaign to next week?" ‚Äî it's in the log. "Why did we stop pursuing that approach?" ‚Äî it's in the log. The act of writing it takes seconds. The value of having it compounds.

HTML dashboards make invisible work visible

Generating a dashboard isn't just about having something pretty to look at. It creates a shareable artefact ‚Äî a URL you can send to a colleague or manager that shows the current state of all work, for all clients, without a meeting. For agency work especially, this changes the conversation from "what are you working on?" to "here's the link."

Getting started

Set it up in under 30 minutes.

You'll need Claude Code (Anthropic's CLI tool) and a GitHub account. The rest is just files.

Create your project folder

Make a folder called 00-daily-indaba (or any name you prefer) inside your Claude workspace. This is where all your daily files will live.

Copy the CLAUDE.md file

The CLAUDE.md file is the heart of the system. It defines the session startup protocol, all shortcuts, the daily file structure, and the dashboard workflow. Copy it into your folder and Claude will read it automatically from the next session.

Initialise a GitHub repo

Run git init in the folder, create a matching GitHub repository, and enable GitHub Pages on the main branch. This gives your dashboards a live public URL.

Start your first daily file

Open Claude Code in the folder and add your first task. Claude will create a YYYY-MM-DD.md file with the correct structure. Start typing your tasks ‚Äî the system takes over from there.

Customise for your clients and workflow

Edit the CLAUDE.md to reflect your client names, task categories, and working style. Add shortcuts that match how you think. The system is yours to shape.
